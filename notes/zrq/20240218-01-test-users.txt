    Target:

        Diagnose issues with test user shares.
        Previous deployment detected errors when importing test users.
        See: notes/zrq/20240213-01-bash-dash.txt
        See: notes/zrq/20240213-04-test-users.txt
        See: notes/zrq/20240214-01-test-users.txt

        I've included the resulting JSON file in the notes for reference.
        See: notes/zrq/20240218-02-test-users.json

        Notes from 20240213 and 20240214 suggest we have a couple of systematic errors.
        1) 30% of the CephFS shares fail to mount properly.
        2) Importing the first set of notebooks reports an IO error.
           However, the notebooks are actually imported in the end.

        The CephFS error message suggested the mount failed because the service
        was slow or overloaded.

        This is a re-test 4 days later to see if the problem with mounting the
        CephFS shares has majically fixed itself.

    Result:

        The stuck deleting shares have been fixed.
        https://github.com/wfau/gaia-dmp/blob/master/notes/zrq/20240213-02-stuck-shares.txt
        https://ucam-rcs.atlassian.net/servicedesk/customer/portal/4/HPCSSUP-66755

        The problems mounting CephFS shares has majically fixed itself.

        Importing the first set of notebooks still reports an IO error.
        However, the notebooks are actually fine.

        So no actual errors then :-)
        Just flakeyness :-(
